第3回 はじめてのPython#

連絡事項・課題（4/* 更新）

この章で学ぶこと#

この章では、数値や文字列といった基本的なデータの扱い方を学ぶ。

今の段階では細かいところはわからなくても、なんとなくPythonを実行できれば十分である。プログラミングは手で覚えるところがあるので、簡単な例も逐一タイピングして、実際に動かしながら学んでほしい。

最初のプログラム#

まずはPythonプログラムを動かしてみよう。

[1]:

print("Hello, world!")

Hello, world!

ノートブックの文字を入力できる部分をセルという。 セルに見本通りに文字を入力して、左の実行ボタンを押すと Hello, world! と表示されるはずだ。 この Hello, world! と表示させるプログラムは、どのプログラミング言語でも最初の一歩として紹介されることが多い。

前回の復習になるが、このようなコンピューターに対する命令のことをプログラムといい、それをPythonなどの言語で記述したものをソースコード （またはコード）という。 また、左の実行ボタンを押す以外の実行方法として、キーボードで Shift を押しながら Enter と入力する方法がある。便利なので覚えておこう。

半角と全角

上のコードを入力する際に、全角文字を使うとその場所によってはエラーになる。あとで説明するように日本語の文字列データを扱うときや、コメントを書くとき以外は、全角文字を使うことはほぼない。 常に半角文字になっているか気をつけながら入力する習慣をつけよう。

計算の順番#

次のコードの実行結果を予想してみよう。

[8]:

2 + 3 ** 2 / 4 * 2 ** 2 - 5

[8]:

6.0

予想は当たっただろうか。ここでは計算の順番に関するルールを学ぶ。

まず、演算子には優先順位があり、累乗 **、掛け算・割り算 *, \, \\, %、足し算・引き算 +, - の順に優先される。また、同じ優先順位の演算子は、前から順に実行される。ただし、普通の数学と同じく () を付ければ、その中身の計算が最優先される。

このルールにしたがうと、上の数式は次のように計算される。
まず、足し算・引き算より掛け算・割り算・累乗が優先されるので、真ん中の項が先に計算される。() を使って明示的に書けば、次の式と同じということである。
[9]:

2 + (3 ** 2 / 4 * 2 ** 2) - 5

[9]:

6.0

さらに、真ん中の項のうち累乗が優先されるので、次のように 3**2 と 2**2 が先に計算される。

[10]:

2 + ((3 ** 2) / 4 * (2 ** 2)) - 5

[10]:

6.0

累乗を計算したあとの 9 / 4 * 4 は前から順に計算されるので 9 / 4 * 4 = 2.25 * 4 = 9となる。したがって、最終的な答えは 2 + 9 - 5 = 6 となる。

ところで、最初の式はとてもわかりづらいと感じなかっただろうか。 これは長年プログラミングに触れている人にとっても同じであり、最後の()を活用した式の方が見やすい。 間違いを減らすためにも()を活用しながら、わかりやすい記述を心がけよう。 さらにコメントすれば 9 / 4 * 4 より 9 * 4 / 4 の順番の方が、計算の順番に安心感がある。 したがって、最初の式は次のように書くとより良い。

[11]:

2 + ((3 ** 2) * (2 ** 2) / 4) - 5

[11]:

6.0

変数#

ここでは、数値などのデータに名前をつける方法について学ぶ。

先ほどの練習1の答えはそれぞれ次のとおりであった。

[12]:

80 * 6 + 120 * 9

[12]:

1560

[13]:

80 * 0.95 * 6 + 120 * 9

[13]:

1536.0

こういう式を書くとき、数値と演算子のみで書くと、目で見て意味がとりづらくなる。 また繰り返し現れる数値があるが、それぞれ2回タイピングするのは間違いのもとである。

そんなときは数値に名前をつけると便利である。Pythonでは次のような形式でデータに名前をつけることができる。

名前 = 式

左辺の名前のことを変数という。右辺はあえてデータではなく式と表現した。式の場合はそれを評価した結果の値に名前がつく。

変数を使うと、練習1の答えは次のように求めることができる。

[14]:

apple = 80
orange = 120

apple * 6 + orange * 9

[14]:

1560

[15]:

apple * 0.95 * 6 + orange * 9

[15]:

1536.0

先ほどと比べると、式の意味が読み取りやすくなっている。一度変数を定義すれば、他のセルからもその変数を使うことができる。 また変数の値は上書きすることもできる。

[16]:

apple = 100
apple * 6 + orange * 9

[16]:

1680

さて、先ほど右辺を式と表現したが、例えば次のような定義の仕方もできる。

[17]:

x = 1 + 1
x

[17]:

2

次の結果は、やや驚きかもしれない。

[18]:

x = 2
x = x + 1
x

[18]:

3

この実行結果の理由は次のとおりである。x = x + 1 のコードにおいて、まず右辺の式が評価される。x の指す値は 2 であるから、x + 1 の値は 3 である。 したがって、x = x + 1 のコードは x = 3 のコードに読み替えられる。その結果として、x は新しい値 3 の名前として上書きされる。数学とは違い x = x + 1 は方程式ではないので注意しよう。

プログラミングでは、このような計算式がよく出てくるので省略記法もある。具体的には、x = x + 1 は x += 1 と書くことができる。

以下のコードを、結果を予想しながら実行してみよう。

[19]:

x = 5
x = 2 * x + 1
x

[19]:

11

[20]:

x = 9
x /= 3
x

[20]:

3.0

変数名の付け方#

変数の名前は自由につけることができるが、いくつかルールがある。

  1. 使える文字はアルファベット、数字、アンダースコア（ _ ） のみである。

  2. 変数の先頭に数字は使えない。

  3. for や if などの文法的に意味のある単語（予約語）は、そのまま単体では使えない。

例えば、price_apple は変数名にできるが、price-apple はエラーになる。a1 は変数名にできるが、1a はエラーになる。 また、if は使えないが、value_if_a は変数名にできる。

変数名はなるべく英語で表現するのが理想的だが、難しければ日本語のローマ字で表現してもよい。 例えば、みかんは英語で書くと mandarin であるが、日本人にとっては mikan の方がわかりやすいだろう。

文字列#

これまで数値データを見てきたが、文字もデータとして扱うことができる。

Hello, world!プログラムを思い出そう。Pythonでは両端をシングルクォーテーション（ ’ ）またはダブルクォーテーション（ ” ）で囲うと、文字列データとなる。ただし両端の記号は合わせる必要がある。

[21]:

'Good morning.'

[21]:

'Good morning.'

文字列の中で ’ または ” を使う場合は、両端の記号をそれとは違うものにする。

[22]:

'She said, "Good morning." '

[22]:

'She said, "Good morning." '

日本語も扱うことができる。

[23]:

"太郎"

[23]:

'太郎'

先ほど学んだ変数を使ってみよう。

[24]:

name = "次郎"
name

[24]:

'次郎'

文字列においても + 演算子を使うことができる。文字列の場合は、連結を表す。

[25]:

greeting = "おはよう、" + name + "さん"
greeting

[25]:

'おはよう、次郎さん'

f文字列#

上の greeting で行ったように、変数と組み合わせて表示用の文字列を加工する操作はよく行われる。 このとき便利なのが f文字列 である。 これについては実際に使っているところを見せるのが早いだろう。

[26]:

greeting = f"おはよう、{name}さん"
greeting

[26]:

'おはよう、次郎さん'

普通の文字列との違いは、先頭に f がついていることである。このとき波括弧 { } で囲った部分は式と見なされ、評価した結果の値に置き換えられる。例えば変数の場合は、それの指し示すデータに置き換えられる。

[27]:

name = "太郎"
greeting = f"おはよう、{name}さん"
greeting

[27]:

'おはよう、太郎さん'

数式の場合の例を次に示す。

[28]:

f"7 割る 3 = {7//3} 余り {7%3}"

[28]:

'7 割る 3 = 2 余り 1'

[29]:

f"3の平方根の値は{3**0.5}である"

[29]:

'3の平方根の値は1.7320508075688772である'

小数の表示を第2位までにするには、{式 : .2f} と書く。

[30]:

f"3の平方根の値は{3**0.5:.2f}である"

[30]:

'3の平方根の値は1.73である'

表示の整形のための記法は他にも色々とある。必要となったときに逐一調べると良いだろう。

エスケープシーケンス#

文字列データで改行を表したいとする。愚直に書くと次のようにエラーになる。

[31]:

message = "こんにちは。
今日の気温は15度です。"

  File "/var/folders/p0/0zrw9jsj03v3vlszs8v3m2p40000gn/T/ipykernel_67230/4098129477.py", line 1
    message = "こんにちは。
                     ^
SyntaxError: EOL while scanning string literal

改行を表現するにはエスケープシーケンスを用いる。まずは次の2つを覚えておけばよい。

エスケープシーケンス

意味

\n

改行

\t

タブ

エスケープシーケンスを用いれば、上の例は次のように修正できる。なお、print()を使わないと\nがそのまま表示されてしまうので、ここではprint()を用いる。

[32]:

message = "こんにちは。\n今日の気温は15度です。"
print(message)

こんにちは。
今日の気温は15度です。

\ と ¥

MacやLinuxにおいて \ に相当する記号は、Windowsでは ¥ になる。Windowsを使っている方は、適宜 \ を ¥ に読み替えてほしい。例えば、改行を表すエスケープシーケンスは、Windowsでは ¥n である。

コードの改行

コードの1行が長くなると、2行に分けて記述したくなることがある。そんなときは文末に\と入力すると、次の行に継続してコードを書くことができる。例えば、

calc = 1 + 2 + \
    3 + 4 + 5
message = "The place is " + \
    "beautiful."

は次のコードと同じ意味になる。

calc = 1 + 2 + 3 + 4 + 5
message = "The place is " + "beautiful."

コメント#

プログラムにはコメントをつけることができる。

# を入力すると、その行のそれ以降の文字はコメントとなり、Pythonの実行対象とならない。

[33]:

# これはコメントです。

r = 5
pi = 3.14
pi * r ** 2  # 円の面積の計算

[33]:

78.5

下のプログラムにコメントを書き加えてみよう。

[34]:

m = (1 + 2 + 3 + 4 + 5) / 5
m

[34]:

3.0

エラーと対処法#

これまでプログラムを実行してきて、何度かエラーに遭遇したのではないだろうか。 エラーの原因をつきとめて修正する作業のことをデバッグという。

ここではよくあるエラーを紹介する。 原因を考えながらエラーメッセージを読み、デバッグしてみよう。

例1: 誤字

[35]:

primt("Hello, world!")

---------------------------------------------------------------------------
NameError                                 Traceback (most recent call last)
/var/folders/p0/0zrw9jsj03v3vlszs8v3m2p40000gn/T/ipykernel_67230/1762608797.py in <module>
----> 1 primt("Hello, world!")

NameError: name 'primt' is not defined

エラーメッセージには、primtという変数は定義されていないと書かれている。 このような場合は、タイピングミスをしていないか確認しよう。

例2: 全角文字

[36]:

print("Hello, world!"）

  File "/var/folders/p0/0zrw9jsj03v3vlszs8v3m2p40000gn/T/ipykernel_67230/1922743172.py", line 1
    print("Hello, world!"）
                         ^
SyntaxError: invalid character '）' (U+FF09)

エラーメッセージには、無効な文字 ‘）’ があると書かれている。このような場合は、全角文字を使っていないか確認しよう。

全角文字にまつわるエラーは、目で見てもよくわからないことがあるので厄介である。 次のエラーの原因がわかるだろうか？

[37]:

print("Hello, world!")

  File "/var/folders/p0/0zrw9jsj03v3vlszs8v3m2p40000gn/T/ipykernel_67230/2817388938.py", line 1
    print("Hello, world!")
                          ^
SyntaxError: invalid non-printable character U+3000

例3: インデント

[38]:

 message = "Hello, world!"
    print(message)

  File "/var/folders/p0/0zrw9jsj03v3vlszs8v3m2p40000gn/T/ipykernel_67230/869639648.py", line 2
    print(message)
    ^
IndentationError: unexpected indent

エラーメッセージには、予期せぬインデントがあると書かれている。行頭に空白文字を入れて、字下げを行うことをインデントと言う。 Pythonではインデントも文法の一部であり、後々 if 文や、for 文などブロックを伴う構文を書くときに重要な役割を果たす。 しばらくはインデントは使わず、左端を揃えてコードを書くようにしよう。

エラーが解決できなかったら？

もし自力で解決できそうにないエラーに遭遇したら、エラーメッセージを検索してみよう。多くの場合、解決のためのヒントが見つかるだろう。
